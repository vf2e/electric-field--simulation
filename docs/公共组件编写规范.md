# QML 公共组件使用规范

适用范围：所有使用 `qml/common/` Flat 组件体系的 Mag-aim 桌面产品。

**视觉与交互标准** → [`UI_DESIGN_SYSTEM.md`](./UI_DESIGN_SYSTEM.md)

## 1. 统一原则

- 页面业务逻辑可以不同，视觉基础层必须统一。
- 在面板模块中，默认不直接写原生 `Button` / `TextField`。
- 优先使用 `qml/common/` 目录下组件：
  - `FlatButton.qml`
  - `FlatTextField.qml`
  - `FlatSliderRow.qml`
  - `FlatToggle.qml`
  - `FlatCard.qml`
  - `FlatDivider.qml`
  - `FlatIconButton.qml`
  - `FlatSectionLabel.qml` — 分区小标题（「显示选项」「色标范围」）
  - `FlatModalScrim.qml` — 可动画模态遮罩

模态弹窗（设置、消息）统一挂载在 `qml/overlays/AppOverlay.qml`，**不得**放在侧栏子组件内。

## 2. 组件选型规则

- `FlatButton`
  - `tone: "primary"`：主操作（打开、完成、确认）
  - `tone: "secondary"`：次操作（设置、重置、取消）
  - `tone: "danger"`：危险操作（删除、停止）
- `FlatTextField` — 所有单行数值/文本输入
- `FlatSliderRow` — 标签 + 滑条 + 数值
- `FlatToggle` — 布尔开关
- `FlatCard` — 卡片容器；QQuickWidget 内 `useShadow: false`
- `FlatSectionLabel` — 11px 加粗分区标题

## 3. 主题与 themeColors

每个面板通过 `App.themePalette(App.themeIndex)` 注入：

```qml
readonly property var themePalette: App.themePalette(App.themeIndex)
themeColors: themePalette
```

### 3.1 Qt 5.15 禁止覆盖的属性

| 禁止 | 替代 |
|------|------|
| `palette` | `themeColors` |
| 自定义 `checked`（AbstractButton） | `checkable: true` + 基类 `checked` |
| `cursorShape` | 不设置 |
| `enabled`（Item 子类冲突） | `sliderEnabled` 等语义化名称 |

## 4. 模态弹窗约定

```qml
// 侧栏按钮 — 触发全屏居中弹窗
FlatButton {
    text: qsTr("设置")
    onClicked: App.openSettings()
}

// AppOverlay.qml — 监听并打开
Connections {
    target: App
    function onSettingsRequested() { settingsDialog.open() }
    function onErrorOccurred(message) { messageDialog.show(qsTr("错误"), message) }
}
```

弹窗打开/关闭时调用 `App.notifyModalVisible(visible)`，以便 C++ 层切换鼠标穿透。

## 5. 自检清单

- [ ] 无侧栏内居中的 SettingsDialog / MessageDialog
- [ ] 无原生 `Button` / `TextField` 手写样式
- [ ] 主次危险按钮 `tone` 正确
- [ ] 新 QML 已加入 `resources/app.qrc`
- [ ] 色值来自 `themeColors`，无硬编码语义色

## 6. 扩展建议

- 第三次以上重复样式 → 提取 `FlatXxx.qml` 到 `common/`
- 参考 [`UI_DESIGN_SYSTEM.md`](./UI_DESIGN_SYSTEM.md) 第 10 节走查清单
